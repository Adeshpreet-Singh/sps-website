## 6. Next.js 16 Metadata Patterns

### Current Implementation

```tsx
// src/app/layout.tsx
export const metadata: Metadata = {
  title: "SPS Installation ‚Äî Appliance Installation & Plumbing | Metro Vancouver",
  description: "Metro Vancouver's trusted partner for professional appliance installation and plumbing services ‚Äî serving both residential homes and commercial properties.",
};
```

### Findings

| # | Severity | Issue | Detail |
|---|----------|-------|--------|
| 1 | **Good** | Root metadata defined in layout.tsx | Correct Next.js 16 pattern ‚Äî metadata cascades to all child pages. |
| 2 | **Medium** | No per-page metadata | Only `layout.tsx` defines metadata. All pages share the same title and description. This hurts SEO ‚Äî each page should have unique metadata. |
| 3 | **Medium** | No `metadataBase` set | Without `metadataBase`, Open Graph URLs are relative. Should be set to `https://spsinstallation.com`. |
| 4 | **Medium** | No Open Graph metadata | No `openGraph` object defined. Social sharing (Facebook, LinkedIn, Slack) will show generic previews. |
| 5 | **Medium** | No Twitter/X card metadata | No `twitter` object defined. |
| 6 | **Low** | No `robots` metadata | Defaults are fine (index, follow), but explicit is better for complex sites. |
| 7 | **Low** | No `icons` metadata | No favicon or apple-touch-icon configured in metadata. |
| 8 | **Info** | No `generateMetadata` functions | For a static site, this is fine ‚Äî but per-page metadata would improve SEO significantly. |

### Recommendations
- **Add `metadataBase`** to layout.tsx: `metadataBase: new URL("https://spsinstallation.com")`
- **Add per-page metadata** using `export const metadata` or `export function generateMetadata()` in each page.tsx:
  - `/about` ‚Äî "About SPS Installation | Metro Vancouver's Trusted Installers"
  - `/services` ‚Äî "Our Services | Appliance Installation & Plumbing | SPS"
  - `/services/appliance-installation` ‚Äî "Appliance Installation Services | SPS Installation"
  - `/services/plumbing` ‚Äî "Plumbing Services | SPS Installation"
  - `/services/residential` ‚Äî "Residential Installation Services | SPS Installation"
  - `/services/commercial` ‚Äî "Commercial Installation Services | SPS Installation"
  - `/reviews` ‚Äî "Customer Reviews | SPS Installation"
  - `/service-area` ‚Äî "Service Area | Metro Vancouver & Lower Mainland"
  - `/contact` ‚Äî "Contact Us | Get a Free Quote | SPS Installation"
- **Add Open Graph** metadata for social sharing.
